1. Project Overview & Objectives

This project, named Eduwide (commercially branded as EduLK), is a cross-platform mobile application developed using Flutter. The system serves as a secure, scalable authentication and profile management portal. It incorporates a role-based access control (RBAC) model to distinguish between Admin and User accounts. Additionally, it integrates with Cloudinary CDN for remote media storage, ensuring efficient handling of profile pictures without stressing the primary database.

Key Objectives:

  • Implement secure sign-up, sign-in, and password recovery mechanisms using Firebase.
  • Enable detailed profile management by linking Firestore documents to Auth accounts.
  • Implement UI dispatching based on server-side roles (Admin vs. User).
  • Incorporate direct HTTP multipart uploads to Cloudinary storage buckets.
  • Add email verification checks that restrict profile actions until the email link is verified.
  • Apply custom timezone database localization (Colombo, Sri Lanka).

4. Architectural Workflow & Router Control

The application employs a stream listener inside the entry file to monitor authentication state transitions. Once authenticated, the DashboardWrapper executes a Firestore lookup to retrieve the current user's role metadata.

1. main() Initializes -> Configures TimeZone (Asia/Colombo) -> Initializes Firebase.
2. AuthWrapper listens to authStateChanges Stream.
   ├── ConnectionState.waiting -> Shows FirebaseLoadCheckScreen
   ├── Snapshot has no data    -> Redirects to StartPage
   └── Snapshot has user data  -> Redirects to SimplePreloadScreen -> DashboardWrapper
3. DashboardWrapper retrieves current Firebase UID.
   └── Reads Firestore collection `/users/{uid}`.
       ├── role == "admin"     -> Launches AdminDashboard
       └── role == "user"      -> Launches UserDashboard

5. Database Schema Definition (Cloud Firestore)

The app stores non-credential user records in Firestore under the users collection. Each record maps directly to the user's Auth uid:

Field Name Data Type Database Purpose
uid String Primary key corresponding to the Firebase Auth ID.
name String The full name inputted by the user during signup.
email String Primary communication email address.
nic String National Identity Card number for identity verification.
mobile String Mobile telephone contact details.
role String Defines privileges (restricted options include "admin" or "user").
profileImageUrl String The network URL retrieved from Cloudinary after uploading.

6. Core Technical Code Components

6.1 Firestore Profile Serializer

The model class AppUser converts Firestore data mapping keys into structured Dart objects and vice versa:

class AppUser {
  final String uid;
  final String email;
  final String name;
  final String nic;
  final String mobile;
  final String role;

  AppUser({
    required this.uid,
    required this.email,
    required this.name,
    required this.nic,
    required this.mobile,
    required this.role,
  });

  Map<String, dynamic> toMap() {
    return {
      'uid': uid,
      'email': email,
      'name': name,
      'nic': nic,
      'mobile': mobile,
      'role': role,
    };
  }

  factory AppUser.fromMap(String uid, Map<String, dynamic> map) {
    return AppUser(
      uid: uid,
      email: map['email'] ?? '',
      name: map['name'] ?? '',
      nic: map['nic'] ?? '',
      mobile: map['mobile'] ?? '',
      role: map['role'] ?? 'user',
    );
  }
}

6.2 Direct Cloudinary HTTP Upload

To keep the application thin, profile photos are posted to Cloudinary via a direct HTTP Multipart Post request. The Cloudinary server returns a JSON payload containing the public secure URL:

Future<String?> _uploadToCloudinary(File file) async {
  final uri = Uri.parse("https://api.cloudinary.com/v1_1/dqeptzlsb/image/upload");
  final request = http.MultipartRequest("POST", uri);
  
  request.fields['upload_preset'] = 'flutter_mediq_upload';
  request.files.add(await http.MultipartFile.fromPath('file', file.path));

  final response = await request.send();
  if (response.statusCode == 200) {
    final responseData = await response.stream.toBytes();
    final responseString = String.fromCharCodes(responseData);
    final jsonMap = jsonDecode(responseString);
    return jsonMap['secure_url']; // Remote URL
  }
  return null;
}
